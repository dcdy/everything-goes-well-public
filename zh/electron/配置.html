安装 electron-vite ​

地址：https://cn-evite.netlify.app/

建议

安装版本（不是最新版，就按照第二步在下面更新，或者手动安装）：

我的版本是 electron-vite2.1.0，electron29.1.4，node20.11.28

shell
pnpm create @quick-start/electron
1

更新包 ​

对比检查是否是最新的包

js
pnpm outdated
1

更新为最新包（如果有的包执行命令后没有更新，就手动改 package.json 对应依赖的版本后执行）

js
pnpm install
1

配置渲染进程 src 路径简写为@ ​

electron.vite.config.js文件

import { resolve } from 'path' 改为 import path from 'path'

js
renderer: {
    resolve: {
      alias: {
        '@': path.resolve('src/renderer/src'),
        '@static': path.resolve('static'),
        '@renderer': path.resolve('src/renderer')
      },
      //extensions数组的意思是在import组件的时候自动补全.vue等文件后缀
      extensions: ['.mjs', '.js', '.ts', '.jsx', '.tsx', '.json', '.vue', '.scss']
    },
}

Path Intellisense 插件配置

  1. 右键点击扩展配置

  3. 添加

    json
    // Path Intellisense路径插件配置
"path-intellisense.mappings": {
    "@": "${workspaceRoot}/src/renderer/src",
    "/": "${workspaceRoot}/",
    "@static": "${workspaceRoot}/static"

环境变量 ​

1.新建.env 文件 ​

最外层新建.env.development 和.env.production 文件

MAIN_VITE 开头是主进程可用的环境变量

PRELOAD_VITE 开头是预加载脚本可用的环境变量

RENDERER_VITE 开头是渲染进程可用的环境变量

自己创建 VITE_ 开头的环境变量需要在 electron.vite.config.ts 文件中配置envPrefix: 'VITE_'

shell
## .env.development
VITE_BASE_URL="http://192.168.2.59:8569"
VITE_APP_TITLE="Dev"
MAIN_VITE_SOME_KEY="1"    ## 主进程可用
PRELOAD_VITE_SOME_KEY="2"   ## 预加载脚本可用
RENDERER_VITE_SOME_KEY="3"  ## 渲染器可用

/// <reference types="vite/client" />

interface ImportMetaEnv {
    readonly VITE_BASE_URL: string;
    readonly VITE_APP_TITLE: string;
    readonly MAIN_VITE_SOME_KEY: string;
    readonly PRELOAD_VITE_SOME_KEY: string;
    readonly RENDERER_VITE_SOME_KEY: string;
    // more env variables...
}

interface ImportMeta {
    readonly env: ImportMetaEnv;
}

3.使用(import.meta.env.VITE_BASE_URL) ​

import.meta.env 有一些默认值

js
BASE_URL: '/'; // 这个值通常等于你在 vite.config.js 中配置的 base 属性的值，或者默认为 '/'。你可以这样使用它：<script src="${import.meta.env.BASE_URL}my-script.js">。
DEV: true; // 布尔值，当应用在 'development' 模式下运行时，这个值为 true，否则为 false。
MODE: 'development'; // 应用的当前模式 （'development'，'production' 或 'test'）
PROD: false; // 布尔值，当应用在 'production' 模式下运行时，这个值为 true，否则为 false。
SSR: false; // 布尔值，表示当前的代码是否在服务器端渲染（Server-Side Rendering，SSR）的上下文中执行。如果代码在服务器端运行，import.meta.env.SSR 的值为 true，否则为 false。
