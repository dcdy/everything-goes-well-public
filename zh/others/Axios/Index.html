Axios ​

官网：Axios 中文文档 | Axios 中文网 (axios-http.cn) ​

一个基于promise的网络请求库

是isomorphic 的(即同一套代码可以运行在浏览器和 node.js 中)。在服务端它使用原生 node.js http 模块, 而在客户端 (浏览端) 则使用 XMLHttpRequests。

特点： ​

  • 不同环境不同实现：nodejs( http 请求)、浏览器( XMLHttpRequests)
  • 基于 Promise 也支持 Promise API
  • 拦截请求和响应
  • 转换请求和响应数据
  • 取消请求
  • 超时处理
  • 查询参数序列化支持嵌套项处理
  • 自动将请求体序列化为：
    • JSON (application/json)
    • Multipart / FormData (multipart/form-data)
    • URL encoded form (application/x-www-form-urlencoded)
  • 将 HTML Form 转换成 JSON 进行请求
  • 自动转换 JSON 数据
  • 获取浏览器和 node.js 的请求进度，并提供额外的信息（速度、剩余时间）
  • 为 node.js 设置带宽限制
  • 兼容符合规范的 FormData 和 Blob（包括 node.js）
  • 客户端支持防御XSRF

模块化安装： ​

bash
npm install axios

bower install axios

yarn add axios

axios 原生写法 ​

1. GET 请求 ​

使用场景：

  • 获取用户信息、商品列表、文章详情等数据查询操作
  • 搜索功能的实现
  • 页面初始化数据的加载

优点：

  • 请求可以被浏览器缓存
  • 可以添加到浏览器收藏夹
  • 参数直观，便于调试

缺点：

  • URL 长度受限制
  • 参数暴露在 URL 中，不适合传输敏感数据
  • 参数类型受限，无法传输复杂数据结构
js
axios({
    method: 'get',
    url: process.env.VUE_APP_BASE_API + '/api/get',
    headers: {},
    params: params // 参数会自动转换为查询字符串
})
    .then((res) => {
        console.log('获取数据成功');
    })
    .catch((err) => {});

2. POST 请求 ​

使用场景：

  • 用户登录注册
  • 提交表单数据
  • 上传文件
  • 创建新的资源

优点：

  • 可以传输大量数据
  • 支持多种数据格式（JSON、FormData 等）
  • 数据在请求体中，相对安全

缺点：

  • 不能被浏览器缓存
  • 相比 GET 请求消耗更多服务器资源
js
axios({
    method: 'post',
    url: process.env.VUE_APP_BASE_API + '/api/post',
    headers: {},
    data: params // 数据在请求体中传输
})
    .then((res) => {
        console.log('数据提交成功');
    })
    .catch((err) => {});

3. PUT 请求 ​

使用场景：

  • 更新用户完整信息
  • 替换服务器上的资源
  • 需要幂等性的完整更新操作

优点：

  • 具有幂等性，多次请求结果一致
  • 语义明确，表示完整替换

缺点：

  • 需要传输完整的资源数据
  • 不适合部分更新
js
axios({
    method: 'put',
    url: process.env.VUE_APP_BASE_API + '/api/update',
    headers: {},
    data: params
})
    .then((res) => {
        console.log('更新数据成功');
    })
    .catch((err) => {});

4. DELETE 请求 ​

使用场景：

  • 删除用户账号
  • 删除文章或评论
  • 取消订单
  • 清除缓存

优点：

  • 语义明确
  • 幂等性，多次删除结果一致
  • 一般只需要传递资源 ID

缺点：

  • 操作不可逆
  • 需要严格的权限控制
js
axios({
    method: 'delete',
    url: process.env.VUE_APP_BASE_API + '/api/delete',
    headers: {},
    params: params // DELETE请求通常使用params传参
})
    .then((res) => {
        console.log('删除数据成功');
    })
    .catch((err) => {});

5. PATCH 请求 ​

使用场景：

  • 修改用户部分信息
  • 更新文章部分内容
  • 状态标记的修改

优点：

  • 只需要传输需要更新的字段
  • 节省带宽

  • 不具有幂等性
  • 部分服务器可能不支持
js
axios({

    headers: {},
    data: params
})
    .then((res) => {
        console.log('部分更新数据成功');
    })
    .catch((err) => {});

6. HEAD 请求 ​

使用场景：

  • 检查资源是否存在
  • 获取文件大小等元信息
  • 检查资源是否更新

优点：

  • 不返回响应体，节省带宽
  • 快速获取资源信息
  • 适合预检请求

缺点：

  • 只能获取响应头信息
  • 使用场景相对较少
js
axios({
    method: 'head',
    url: process.env.VUE_APP_BASE_API + '/api/check',
    headers: {}
})
    .then((res) => {
        console.log('检查资源状态成功');
    })
    .catch((err) => {});

7. OPTIONS 请求 ​

使用场景：

  • CORS 预检请求
  • 检查服务器支持的请求方法
  • 检查服务器功能是否可用

优点：

  • 用于跨域请求的预检
  • 可以探测服务器能力
  • 有助于接口文档生成

缺点：

  • 增加额外的网络请求
  • 主要用于浏览器自动发起
js
axios({
    method: 'options',
    url: process.env.VUE_APP_BASE_API + '/api/options',
    headers: {}
})
    .then((res) => {
        console.log('获取服务器支持的HTTP方法');
    })
    .catch((err) => {});

8. 并发请求 ​

使用场景：

  • 页面初始化需要多个接口数据
  • 多个独立但相关的数据请求
  • 需要同时处理多个异步操作

优点：

  • 提高请求效率
  • 避免回调地狱
  • 统一的错误处理

缺点：

  • 任一请求失败都会触发 catch
  • 需要合理控制并发数量
  • 增加代码复杂度
js
// 同时发起多个请求
axios
    .all([
        axios({
            method: 'get',
            url: process.env.VUE_APP_BASE_API + '/api/data1'
        }),
        axios({
            method: 'get',
            url: process.env.VUE_APP_BASE_API + '/api/data2'
        })
    ])
    .then(
        axios.spread((res1, res2) => {
            console.log('所有请求完成');
        })
    )
    .catch((err) => {
        console.log('某个请求失败');
    });
