Vue2 升级 Vue3 ​

使用 GoGoCode 工具转换 ​

重点

官方介绍：https://github.com/thx/gogocode/blob/main/README-cn.md

Vue2 到 Vue3 升级插件：https://github.com/thx/gogocode/tree/main/packages/gogocode-plugin-vue

Element to Element Plus 升级插件：https://github.com/thx/gogocode/tree/main/packages/gogocode-plugin-element

注意

  1. 代码内不能有反义符号`
  2. 代码内不能有import.meta.env

安装 ​

shell
npm install gogocode-cli -g
1

新建空文件夹，初始化 gogocode 工具 ​

shell
gogocode init
npm uninstall sourcemap-codec
npm install @jridgewell/sourcemap-codec --save
1
2
3

使用 ​

Vue2 升级为 Vue3 插件命令 ​

注意

其中./src 是要转换的文件夹，./src-out 是转换完成后输出的文件夹，如果都写成./src，就会替换原文件

路径也可以直接写成某一个指定文件的位置，如./src/Index.vue

1.指定应用规则

添加参数 -p include-rules=xxx,yyy，举例只应用 vModel 及 customDirectives 规则：

shell
gogocode -s ./src -t gogocode-plugin-vue -o ./src-out -p include-rules=vModel,customDirectives
1

规则名称请参看 rules.js 里面的 name 字段。

2.排除应用规则

添加参数 -p exclude-rules=xxx,yyy，举例只排除 vModel 及 customDirectives 规则：

shell
gogocode -s ./src -t gogocode-plugin-vue -o ./src-out -p exclude-rules=vModel,customDirectives
1

建议

排除 eventsApi，event 事件操作安装使用 mitt 代替

shell
gogocode -s ./src -t gogocode-plugin-vue -o ./src-out -p exclude-rules=eventsApi
1

ElementUI 升级为 Element Plus 插件命令 ​

注意

指定应用规则和排除应用规则方式同上

完整规则如下：

建议

排除 icon，图标自己手动改，当在 vue2 中的<i>标签内有其他事件或者样式时，gogocode 转换会直接将这些事件样式去掉，如果本身的<i>标签内就没有事件和样式，那可以不用排除 icon

html
// vue2

解决方案 1：回退路由版本 ​

"vue-router": "~4.1.3",

vue-router 回退版本到 4.1.3 之前

params 改为 query

注意

js

    name: 'Login',
    params: {

        data: 1
    }
});
